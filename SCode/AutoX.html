AutoX

Autonomous Delivery App

Overview

AutoX is a startup that develops self-driving vehicles, providing food & grocery delivery services in the bay area. I had an opportunity to work with the team to redefine a few flows for their mobile app. Additionally, I explored on creating a visual system and also designed the layout of a publication. The app is now live on the App Store and Google Play.

Contract Designer | AutoX Inc

Interaction Design, Visual Design, User Research, Prototyping, Graphic Design

Jul—Nov 2018

Challenges

• Reinvent a clear navigation paradigm for the AutoX mobile app.
• Inform users in the best possible way if their addresses are not in the service area
• Educate the users about picking up from a self-driving car
• Buid trust with users by providing real-time order status information
• Create a visual design system to promote UX and brand consistency

Research

Before tackling any of the challenges above, I decided to give the object-oriented-UX methodology a try. I created an object map main objects(in yellow) and pieces around it laid out. I was intrigued by how this method is able to display a clear picture of how different aspects of the app can be connected. Based on the object map, I built out the information architecture and further defined the navigation.

Iteration & Validation

Before tackling any of the challenges The iteration phase was a collaborative process between the engineers, the product team, and myself. We discussed priorities and feasibility based on the wireframes I created. After a few rounds of quick reviews, I started to design in Sketch based on the feedback I received on the wireframes.

Besides presenting designs, I also joined discussions with the team about product strategy. I did research on competitors and similar products to assess what would make AutoX’s delievery service successful. Due to tight deadlines, we took the approach of shipping the product in short sprints, and make refinements as we receive feedback from users.

“Deliver Now”

One idea came up during our discussion—the app would only require the user to select a delivery date, and the user would tap on the “Deliver Now” button whenever he/she is ready to summon the car. The team believed in this idea and hoped the users would appreciate this new concept of receiving orders. However, results

3 out of 5

State that they might forget to summon the car

4 out of 5

Do not want to wait for the order at home

More Delivery Options

To provide more flexibility to users, the team decided to provide two more delivery options: scheduled delivery (similar to other services), and an on demand option which comes with a drastically higer cost. More options in delivery resulted in a huge challenge on design, escpecially when explaining the differences between each options to the users. I quickly gathered feedback from people who have used grocery delivery services and below are their responses:

5 out of 6

Do not understand the difference between the first two options

4 out of 6

Preferred scheduled delivery

After rounds of internal discussion, the team agreed to keep delivery simple and decided to provide shcedule delivery only.

Order Status

The most unique aspect of this service is that a user would receive his/her order from a self-driving car, which means the user should be able to access the order without any human help. The app aims to provide guidance to the user with visuals and text, allowing first-time user to enjoy the service confidently.

Reffering back to the object map, I designed the order status screen by prioritizing the information that is important to users. Since autonomous delivery service hasn’t been popularized, there is a need to preprare the user for what is about to happen, as well as providing real-time updates about the order.

The first two iterations above have a few missing pieces based on the object map. In order to incorporte all the essential elements, I divided information into a two-step flow for iteration 3. The left screen displays what the user might need to know when the order is en route. The unlock button along with instructions appear when the order has arrived. This design received positive feedback from users as it is much more contextual and only provides information that is necesarry.

Final Prototype

Sign Up

The onboarding experience has been significantly simplified comparing to the previous flow. It would be very frustrating for the user to learn that he/she cannot use AutoX’s delivery service after entering multiple pieces of information and adding a password to create an account. By only obtaining the user’s email address, we are able to inform the user about future updates, and most importantly, providing a more delightful onboarding experience.

Shop

After entering a valid delivery address, the user would land on the shop tab to browse products. AutoX was also planning to add restaurant delivery, but my main focus at the time was to design for the grocery delivery experience.

Check Out

The check out is designed to be quick an simple, with a few necesarry taps to confirm the user’s information. Most of the information will be prefilled if the user is a returning customer.

Below are the steps that a user would need to go through:

1. Confirm products and quantity in cart

2. Browse through a list of available delivery times

3. Edit delivery info if necessary

4. Add a promo code

5. Place an order

Unlock

The most unique aspect of this service is that a user would receive his/her order from a self-driving car, which means the user should be able to access the order without any human help. The app aims to provide guidance to the user with visuals and text, allowing first-time user to enjoy the service confidently.

The user may check the estimated arrivival of the order through the order status page in the app. Another goal we would like to achieve here is to educate the user on how to retrieve the order. When the vehicle arrives at the user’s door front, the app prompts the user to tap on the unlock button to open the trunk of the vehicle.

Visual Design

As I was designign the app, I created a basic visual system with text colors, tint colors, icons and text styles to keep design elements consistent. I made sure the color palette passes accessibility standards of having enough contrast ratio and the text styles matches text styles of iOS’ dynamic type.
